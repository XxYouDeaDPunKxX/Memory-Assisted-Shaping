🧠 Memory-Assisted Shaping

Keep long idea-shaping sessions coherent without turning notes into the work surface.

Memory-Assisted Shaping is a ChatGPT Projects protocol for working on one idea across many turns while preserving decisions, gates, discarded paths, and source boundaries.

The final artifact stays separate from the shaping process.

View on GitHub Use with ChatGPT Projects

🌊 The problem

Long ChatGPT conversations drift.

  • Examples start acting like decisions.
  • Discarded paths return.
  • Open questions disappear.
  • The final output starts carrying process residue.

This is a problem when the conversation is not just a chat, but a shaping session.

⚙️ The model

GPT shapes the idea first.

When continuity would degrade without retention, GPT may emit a small memory signal.

Python appends that signal.

  • GPT does not manage the memory files.
  • GPT does not rewrite the notes.
  • The notes do not become the source of truth.

They only help recover continuity.

🧩 What it tracks

Memory-Assisted Shaping helps preserve:

  • decisions
  • open gates
  • discarded paths
  • source boundaries
  • artifact readiness
  • changes in direction

Only material that affects continuity should be retained.

Ordinary conversation does not become memory.

✅ Use it when

Use it when one idea needs more than a few turns.

  • shaping a protocol
  • designing a workflow
  • clarifying a system
  • preparing a spec
  • refining an implementation handoff
  • keeping a long idea session coherent

🚫 Do not use it when

Do not use it for simple tasks.

  • quick answers
  • ordinary brainstorming
  • general note-taking
  • storing everything
  • replacing judgment with memory
  • treating previous notes as authority

Memory signals preserve context. They do not decide.

📦 Final artifacts

The final artifact is produced only after explicit approval.

It should be clean, standalone, and usable without the messy shaping trace.

It should not contain:

  • raw notes
  • discarded paths
  • process commentary
  • internal reasoning
  • memory signal history

Unless you explicitly ask for an audit-style document.

📁 Files

Core files

  • protocol.md
  • note-io.md
  • notes.py

Generated during use

  • session_meta.json
  • session_notes.jsonl
  • session_export.json

Generated session files are not part of the clean source package.

🚀 Use with ChatGPT Projects

Add the project files to ChatGPT Project Source.

Tell GPT:

Use Memory-Assisted Shaping for this idea.

Then work normally.

Ask for a recap when state matters. Ask for a final artifact only when the shape is ready.
